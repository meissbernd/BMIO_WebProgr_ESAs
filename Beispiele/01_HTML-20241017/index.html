Aufbau eines HTML Dokumentes

Für gute Maschinenlesbarkeit sollte ein Dokument immer korrekt aufgebaut sein. D.h. die Syntax ist korrekt. Aber neben der Syntax ist auch auf die Semantik zu achten.

Die Bestandteile eines Dokumentes sind Elemente, Attribue und Textknoten.

Das gesamte Dokument sollte in sinnvolle Bereiche aufgeteilt werden.

Wichtige Bereiche sind die Navigation, einzelne Artikel, Header und Footer, Listen, Tabellen, Formulare, etc.

Eine Übersicht und die Erklärung zu den einzelnen Elementen finden Sie beim WHATWG, einer Arbeitsgruppe speziell für HTML5 vom W3C.

Syntax

Die korrekte Syntax ist über den Validator nachzuprüfen. Für die meisten Werkzeuge gibt es mittlerweile Plugins, um den Validator zu integrieren.

Elemente und Attribute werden wie folgt notiert.

Elemente mit Inhalt haben ein Start- und ein Endtag. Dazwischen befindet sich der Inhalt. <element>content</element>

Elemente ohne Inhalt nennen sich Standalone Elemente und haben nur ein Starttag. <element> Bis auf wenige Ausnahmen haben diese Elemente i.d.R. Attribute.

Elemente können durch Attribute weiter beschrieben werden. Attribute werden im Starttag notiert. Jedes Element kann beliebig viele Attribute und jedes Attribut beliebig viele Werte besitzen. <element att1="wert1" att2="wert2a wert2b">content</element>

Semantik

Die Semantik, also die Verwendung sinnvoller und bedeutungsvoller Elemente dient vor allem der Barrierefreiheit. Aber auch Wartbarkeit und das Ranking in Suchmaschinen (SEO) profitieren von einer guten Semantik. Die Semantik muß vom Autor geprüft werden.

Einige Semantikfehler findet auch der Validator. Z.B. sollten Artikel immer eine Überschrift haben.

WCAG und ARIA

Mit Aria Attributen können einzelne Elemente besser für Hilfsmittel (z.B. Screenreader) zugänglich gemacht werden. Dabei gilt:

  • Wenn es ein semantisches Element gibt, dann ist dieses zu nutzen.
  • Wenn kein semantisches Element zur Verfügung steht, dann kommt ein ARIA Attribut zum Einsatz.
